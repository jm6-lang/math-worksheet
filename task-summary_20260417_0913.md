### 学到了什么（核心原理）

**5 阶段流水线：**
1. Reconnaissance（侦察）→ 截图 + CSS 提取 + 交互 Sweep
2. Foundation Build → 字体 → 全局 CSS → TypeScript 类型 → SVG → 资源下载
3. Component Spec + Dispatch → 每个 section 写 spec → 分发给 builder
4. Merge + QA → 合并 + 验证 build
5. Visual QA → 像素级对比，禁止跳过

**关键技术点：**
- `getComputedStyle()` 精确提取 CSS 值，禁止估算（"看着像 16px" 是错的）
- 交互模型识别（scroll-driven / click-driven / hover-driven / time-driven）是第 1 优先级
- 每个组件必须有 `.spec.md` 文件，不写 spec 不分发 builder
- spec >~150 行必须拆分（复杂度控制）
- Layered images：背景色 + 前景图 + overlay = 3 个资源都要抓
- 所有状态都要提取，不只是默认状态
